Cinder Coding Standard

This document lays out the formatting guidelines and best practices for working on the Cinder codebase. Please adhere to this standard when submitting code to be included in the library.


Here's a (nonsense) code sample which demonstrates the majority of the formatting guidelines:

  public:public: and private: signifiers indented 2 spaces

    double	calculate2() const;  tab-align groups of member function declarations
        
  private:
    float 	mVarA, mVarB;  class member vars have an m-prefix
    bool	mCached;  tab-align member variable declarations as appropriate
