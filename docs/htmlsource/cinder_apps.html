Understanding Cinder Apps

In general, Cinder users want to create apps. They may create a standard desktop application, an iOS or Windows Store app, a screensaver, or a number of other types of project. Cinder provides a framework for each of these, and more specifically it provides a C++ base class derived from ci::App.


Cinder apps follow a basic structure which we'll look at now. Below is a minimal Cinder application which blanks the screen using OpenGL.


#include "cinder/app/AppBasic.h"
#include "cinder/app/RendererGl.h"
#include "cinder/Vector.h"

using namespace ci;
using namespace ci::app;

class CirclePusherApp : public AppBasic {
  public:
	void	setup() override;
	void	update() override;
	void	draw() override;
	
	Vec2f	mCirclePosition;
};

void CirclePusherApp::setup()
{
	mCirclePosition = Vec2f( 0, getWindowHeight() / 2 );
}

void CirclePusherApp::update()
{
	mCirclePosition += Vec2f( 0.1f, 0 );
}

void CirclePusherApp::draw()
{
	gl::clear();
	gl::color( Vec2f( 1.0f, 0.5f, 0.25f ) );
	gl::drawSolidCircle( mCirclePosition );
}

CINDER_APP_BASIC( CirclePusherApp, RendererGl )

In this code we see an example of a minimal Cinder application called ExampleApp. It is derived from ci::app::AppBasic, which is Cinder's base class for standard desktop applications. This example implements a draw() method which in turn calls gl::clear(), Cinder's routine for clearing the screen using OpenGL.


The last bit of code to notice is the CINDER_APP_BASIC call. This is a macro which cause some additional setup code to be created in order for your app to run. You can think of it as a Cinder's main() routine, and in fact one of the the things the macro creates is your platform's equivalent of the classical C/C++ main(). The second parameter, RendererGl, selects which kind of renderer your application will make use of. In additional to OpenGL Cinder provides Cairo, DirectX and other types of renderers.
